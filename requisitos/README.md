Este diretório organiza os requisitos do sistema por versão. Cada arquivo representa
um ciclo de desenvolvimento com escopo fechado, permitindo rastrear a evolução do
produto ao longo do tempo.

## 📌 Convenção

- Um arquivo por versão, nomeado `vX.Y-descricao.md`
- O status de cada versão é atualizado no índice acima conforme o ciclo avança
- Requisitos **nunca são editados retroativamente** após o início do desenvolvimento — novos requisitos entram sempre num arquivo de versão nova
